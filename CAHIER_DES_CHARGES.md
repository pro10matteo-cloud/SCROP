> Brigade 21 — THE GLITCH / ISEFAC TALKS 2026

## 1. OBJECTIF DU SITE

Le site PSCHITT remplit un **double rôle** :

1. **Vitrine produit / Landing page** — Présenter le spray nasal anti-scroll TikTok de manière immersive, drôle et mémorable. Le visiteur doit comprendre le concept en 3 secondes et avoir envie de scroller (ironie assumée).

2. **Hub de livrables pour le jury** — Centraliser tous les éléments attendus par le hackathon (problématique, benchmark, persona, business plan, storytelling, équipe). Le jury doit pouvoir tout retrouver en naviguant dans le site.

**Format** : One-page scrollable qui raconte une histoire du haut vers le bas :
Le problème → Le produit → Comment ça marche → Pour qui → Le marché → Le business → L'histoire → L'équipe.

## 3. CHARTE GRAPHIQUE

### Palette de couleurs
```
Primaire :     #DFFF00  (Jaune acide)
Fond :         #0A0A0A  (Noir profond)
Accent 1 :    #7B2FBE  (Violet)
Texte clair :  #F5F0EB  (Blanc cassé)
Accent 2 :    #00FFC2  (Vert menthe)
Alerte/CTA :   #FF3333  (Rouge)
```

### Typographies
| Usage | Police | Style |
|-------|--------|-------|
| Titres (H1, H2) | Anton | Bold, condensé, uppercase |
| Corps de texte | Inter | Regular 400, Light 300 |
| Accents / Code | Space Mono | Regular, monospace |

### Principes de design
- **Contrastes forts** — Titres massifs, fond sombre, couleurs vives
- **Asymétrie contrôlée** — Éléments qui débordent, rotations légères, grille cassée
- **Glitch = ennemi** — Tout ce qui représente l'addiction a un effet glitch/distorsion
- **Clean = solution** — Tout ce qui représente PSCHITT est fluide, net, premium
- **Espace négatif** — Sections espacées, le site respire

### Principes d'animation
- **Scroll-driven** — Les animations se déclenchent au scroll, pas au chargement
- **Performantes** — Utiliser `transform` et `opacity` uniquement (GPU-accelerated)
- **Purposeful** — Chaque animation sert le storytelling, pas de déco gratuite
- **Progressive** — Le site reste utilisable sans JS (contenu visible, juste pas animé)

## 4. CONTRAINTES TECHNIQUES

### Performance
- Temps de chargement < 3s sur 4G
- Animations à 60fps (GPU-accelerated)
- Images optimisées (WebP quand possible)
- Lazy loading pour les images sous le fold
- Pas de framework lourd — vanilla JS + GSAP

### Responsive
- **Mobile-first** — La cible principale est sur téléphone
- Breakpoints :
  - Mobile : < 768px
  - Tablette : 768px – 1024px
  - Desktop : > 1024px
- Navigation burger sur mobile
- Cartes en colonne sur mobile, grille sur desktop
- Typographies fluides (clamp)

### Accessibilité (minimum viable)
- Contrastes WCAG AA sur le texte principal
- Alt text sur toutes les images
- Navigation au clavier fonctionnelle
- Structure sémantique (header, main, section, footer)
- Aria-labels sur les éléments interactifs
- `prefers-reduced-motion` respecté

### Déploiement
- Build : aucun — fichiers statiques
- Hébergement : GitHub Pages, Netlify, ou Vercel
- Pas de dépendance serveur
- Un `git push` = déploiement

## 7. CONVENTIONS DE CODE

### Placeholders
Tous les contenus placeholder suivent ce format :
```html
<!-- TODO: [Description du contenu attendu] -->
<span class="placeholder">[PLACEHOLDER : Description]</span>
```

Cela permet de :
- Les repérer visuellement (style CSS distinctif)
- Les retrouver dans le code (`grep "PLACEHOLDER"` ou `grep "TODO"`)
- Les remplacer facilement

### CSS
- Variables CSS dans `:root` pour toutes les couleurs et tailles
- BEM-like naming : `.section-hero__title`, `.card-product__price`
- Mobile-first avec media queries ascendantes
- Classes utilitaires pour les cas simples

### JavaScript
- Vanilla JS, pas de framework
- GSAP + ScrollTrigger pour les animations
- Code organisé par fonction (nav, scroll, animations, easter eggs)
- Event delegation quand possible
